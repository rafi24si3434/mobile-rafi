# PERTEMUAN 12
# IMPLEMENTASI ROOM DATABASE PADA APLIKASI BINA DESA

## Tujuan

Pada pertemuan ini akan diimplementasikan Room Database untuk menyimpan data pengajuan surat secara lokal pada aplikasi Bina Desa.

---

# Konsep Room Database

Room merupakan ORM (Object Relational Mapping) bawaan Android yang digunakan untuk mempermudah pengelolaan database SQLite.

Komponen utama Room:

1. Entity
2. DAO
3. Database

---

# Struktur Package

Buat package baru:

```text
com.example.unluckyyyapps
│
├── data
│   ├── entity
│   │   └── Surat.kt
│   │
│   ├── dao
│   │   └── SuratDao.kt
│   │
│   └── database
│       └── AppDatabase.kt
```

# Langkah 1 - Install Dependency Room

Tambahkan dependency berikut:

```gradle
implementation("androidx.room:room-runtime:2.6.1")
implementation("androidx.room:room-ktx:2.6.1")
ksp("androidx.room:room-compiler:2.6.1")
```

Lakukan Sync Project.

---

# Langkah 2 - Membuat Entity Surat

File:

```text
data/entity/Surat.kt
```

```kotlin
@Entity(tableName = "surat")
data class Surat(

    @PrimaryKey(autoGenerate = true)
    val id: Int = 0,

    val nama: String,
    val jenisSurat: String,
    val keperluan: String,
    val tanggal: String
)
```

Entity ini merepresentasikan tabel surat pada database.

---

# Langkah 3 - Membuat DAO

File:

```text
data/dao/SuratDao.kt
```

```kotlin
@Dao
interface SuratDao {

    @Insert
    suspend fun insert(data: Surat)

    @Delete
    suspend fun delete(data: Surat)

    @Query("SELECT * FROM surat")
    fun getAllData(): LiveData<List<Surat>>
}
```

---

# Langkah 4 - Membuat Database Class

File:

```text
data/database/AppDatabase.kt
```

```kotlin
@Database(
    entities = [Surat::class],
    version = 1
)
abstract class AppDatabase : RoomDatabase() {

    abstract fun suratDao(): SuratDao
}
```

# Langkah 7 - Simpan Data ke Room

Pada tombol:

```text
btnSimpan
```

Tambahkan logika:

```kotlin
btnSimpan.setOnClickListener {

    val surat = Surat(
        nama = etNama.text.toString(),
        jenisSurat = spJenis.selectedItem.toString(),
        keperluan = etKeperluan.text.toString(),
        tanggal = getCurrentDate()
    )

    lifecycleScope.launch {
        database.suratDao().insert(surat)
    }
}
```

# Perubahan Database

Jika Entity berubah:

- Tambah kolom
- Hapus kolom
- Ubah tipe data

Maka:

1. Naikkan version database
2. Tambahkan migration

Atau gunakan:

```kotlin
.fallbackToDestructiveMigration()
```

saat proses development.
